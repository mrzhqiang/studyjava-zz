# 工欲善其事必先利其器
作为程序员，怎么能没有趁手的利器呢？

### 免费VPN
由于众所周知的原因，寻找权威的、全面的、正确的资料，有时候会变得十分困难，甚至完全获得不到。因此需要借助FQ利器，也就是VPN来翻越千山万水，抵达真知的彼岸。
- [x] [蓝灯](https://github.com/getlantern/lantern)：免费、稳定、简单，每台机器每个月500mb流量
- [x] [XX-Net](https://github.com/XX-net/XX-Net)：免费、稳定、简单，每个AppId限制1Gb流量（可自己配置AppId）
- [ ] [其他](https://github.com/search?o=desc&q=VPN&s=stars&type=Repositories&utf8=%E2%9C%93)：热门、与时俱进，github搜索列表

### 高效IDE
“高效”是相对于记事本而言，并不想比较各个工具之间的差异性，也没有嫌弃未列出的IDE。

以下是文本编辑器，适合初学者使用，没有集成复杂的功能
- [x] [Atom](https://atom.io/)：免费，支持多语言、支持git和markdown，需要较高配置
- [ ] [Notepad++](https://notepad-plus-plus.org/)：免费，支持多语言、体积小、运行快，没有配置要求
- [x] [Sublime Text](http://www.sublimetext.com/)：收费，但无限期试用，支持多语言、体积小、运行快，没有配置要求

对于我而言，目前使用的是Atom。作为代码查看工具，它足够漂亮也足够清晰，支持git和markdown，让我像使用IntelliJ IDEA一样身心愉悦。并且当打开一个源代码文件时，它会自动囊括文件所在工程，不用逐个打开浏览。比较适合用来检阅源代码，编辑说明文档，撰写读书笔记。

以下是开发工具，适合项目开发，拥有更高效的能力
- [ ] [Eclipse](https://www.eclipse.org/home/index.php)：强大、快速、体积小
- [x] [IntelliJ IDEA](https://www.eclipse.org/home/index.php)：强大、智能、无与伦比的开发体验

之前使用Eclipse有一段时间，也曾钦佩它的强大之处，但总觉得有所欠缺。直到遇见IntelliJ IDEA，体验过它的智能化，很快就爱上了这个开发工具。

### 版本管理
虽然IntelliJ IDEA已经集成git的相关指令，但相对而言还不是那么强大。并且由于众所周知的原因，用IDEA打开一个很大的工程，会有很长的准备工作，所以不希望在非工作状态下，用IDEA来管理工程的版本。那么，有什么轻量级的工具，适合用来提升版本管理效率呢？

首先列出版本管理工具：
- [x] [Git](https://git-scm.com/)：科学管理、代码追踪、多人协同
- [ ] SVN：并不希望使用，如若坚持，请baidu 或google

接着是简化版本管理命令的工具：
- [x] [TortoiseGit](https://tortoisegit.org/)：图标化、图形化、快捷命令、支持汉化
- [x] [Sourcetree](https://www.sourcetreeapp.com/)：工具化、简单命令、预览改动、支持中文、gitflow

目前已转为使用Sourcetree，首先它图形化比较好，其次支持gitflow工作，再则是它将命令简化得比较清晰，并且支持预览。对于各个分支的进度，也显示为和IntelliJ IDEA一样的树状进度，并且切换分支相对于TortoiseGit更为方便，实在是不可多得的git助手。
