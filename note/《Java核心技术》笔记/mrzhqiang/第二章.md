# Java程序设计环境
这部分内容还是比较适合第一时间看，一边从上手操作中获得乐趣，一边感受JAVA前置工作的轻松。其他语言很难有下载一个JDK安装后，从命令行就能编写并运行第一行代码。

2.1
- 理论部分其实很枯燥，看看就好，最关键的是你能从眼花缭乱的下载网页中，准确地找到你需要的JDK版本——最新的不一定最适合你——因此建议使用JDK1.8，并根据系统是32位还是64位来决定是下载x86还是x64。
- 安装时，虽然书中说，Windows系统最好不要接受默认位置，但实际上开发者有可能频繁重装系统，或使用其他电脑，如能坚持默认位置，这样也方便其他软件设置JDK目录，并且当JDK自身升级时，会有一个并排的安装目录，不至于硬盘中到处都是JDK的影子。
- 设置环境变量是大部分新手都会出问题的地方，原因在于，其一对操作系统不熟，不明白环境变量的意义；其二没有理解操作步骤。关键点是用“;”作为JDK路径的结尾，以及包含空格的路径使用“”括起来。
- 库源文件和文档，对于新手来说具有一定的意义，但熟手基本上使用IDE就能直接查看源码，再配一个对应版本的电子文档，完全不需要对这一步进行操作。

2.2 命令行是神通广大的工具，但是遇到电脑小白，这部分肯定会揪出很多问题。

2.3 集成开发环境（IDE），目前最好的入门级IDE是Eclipse，原因在于它足够小、足够纯粹、足够快，更由于Eclipse不需要太好的电脑配置。当具备一定的开发经验，需要提升效率、提升眼界、提升技术的时候，推荐使用IntellJ IDEA这款神器。

2.4 图形化程序通常不是移动开发方向的JAVA学习者需要深入的部分，玩一玩还差不多。

2.5 关于applet就如同书中所说，你对“过去的记忆”感兴趣，那么你可以继续看下去。
