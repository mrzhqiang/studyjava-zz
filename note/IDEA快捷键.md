# IDEA快捷键
开发久了，有时候双手根本不想离开键盘，更别说去拿鼠标点来点去，那么熟记几个快捷键，对于开发来说会有很大的帮助。

## 代码补全
在Java中，分号`;`是避不开的一道风景线，它的存在会让很多新手犯下不可饶恕的错误。作为现代的开发者，锱铢必较是很有必要的态度，但较真的前提是不浪费时间，而事实上分号是多余的符号，因为它仅仅为了标记一行代码的结束。那么，请熟记`Ctrl + Shift + Entel`快捷键，会让你省去寻找`;`键的时间，并且很多情况下，你并不能准确找到分号键，以至于你要么低头看键盘，要么按错成`l`或`'`。

### 关于`Ctrl + Shift + Entel`的其他作用
- 自动完成括号、花括号的补全，比如输入if，然后按下快捷键，则自动完成如下所示代码：
```
if () {
}
```
这就避免在多重条件判断以及循环控制的情况下，对括号的`Shift+9/Shift+0`以及花括号的`Shift+[/Shift+]`多次按键出错几率。开发者不应该在这些地方浪费时间，感受到沮丧。

### 关于`Ctrl + Space`、`Ctrl + Alt + Space`、`Ctrl + Shift + Space`
- 基础代码补全，即`Ctrl + Space`，通常被Windows系统下的输入法切换，给全局屏蔽掉，要理解它的作用，参照Eclipse的Alt+/键就好
- 智能代码补全，即`Ctrl + Alt + Space`，通常在输入字符的时候，就已经有了提示，但是很多情况下，并不是你输入的代码。为了重新得到提示，按下这个快捷键，你就可以继续嗨皮地工作了。它通常罗列了最接近的变量+方法+常量
- 智能匹配，即`Ctrl + Shift + Space`，一般情况下和智能补全差不多，但它能检测需要的类型，准确定位需要补全的代码，将嗨皮进行到无与伦比的境界。通常情况下，用于方法参数、if/else等逻辑控制语句参数的补全比较合适

## 其他快捷键对应（纯手打）

### 基本应用
- `Ctrl + Q` 查看JavaDoc文档，也就是：
```
/**
 * 这样的注释形式，表示JavaDoc文档
 */
```
- `Ctrl + Z` 撤销编辑，恢复代码；与之对应的是：`Ctrl + Shift + Z` 还原编辑，回档修改
- `Ctrl + W` 选中当前代码块的所有内容，如果重复按下的话，将对代码块的选中层层扩大范围
- `Ctrl + S` 保存修改，虽然IDEA是自动保存修改内容状态的工具，但依然挡不住突然断电的恐惧
- `Ctrl + X` 剪切选中的代码，如果没有选中任何内容，将剪切鼠标光标所在行的一行代码
- `Ctrl + E` 弹出曾经打开的历史文件面板
- `Ctrl + D` 快速复制当前一行代码到下一行
- `Ctrl + F` 查找内容，如果是选中的情况下按下，那么就自动查找选中的内容
- `Ctrl + G` 定位光标到文件的某行/某列
- `Ctrl + B` 跳转到光标所在类、接口等源码文件
- `Ctrl + Y` 删除整行代码
- `Ctrl + H` 打开当前类的层次结构窗口
- `Ctrl + N` 搜索类文件
- `Ctrl + U` 进入父类或父类的方法
- `Ctrl + J` 提供输入建议，比如一些很屌的代码缩写（一种只需要简单的字母组合就可以得到模板代码的方式）
- `Ctrl + M` 将鼠标光标所在位置，移到编辑窗口的中间
- `Ctrl + I` 列出可以实现的接口方法
- `Ctrl + O` 列出可以重写的父类方法
- `Ctrl + P` 列出方法参数的情况，包括所有重载的方法
- `Ctrl + L` 查找内容时，定位到下一个查找的内容，与F3的功能一样

### 深入探索
- `Ctrl + Alt + C` 将所选内容或当前一行的代码，变成常量的形式，如果可以的话
- `Ctrl + Alt + V` 将所选内容变成变量的形式，通常是对有返回值的方法进行操作
- `Ctrl + Alt + M` 将所选内容转为静态方法，也就是将逻辑进行重构
- `Ctrl + Alt + T` 提供一系列的模板代码，省却了手动输入的麻烦
- `Ctrl + Alt + ←/→` 导航到光标的上一次/下一次位置
- `Ctrl + Alt + L` 快速格式化当前类文件的代码
- `Ctrl + Shift + F` 查找包含字符串的所有文件
- `Alt + F7` 查找类的使用情况
- 双击`Shift`查找工程的所有文件

## 总结
基本上用到的快捷键都已经罗列在上，只要花点时间掌握这些快捷键，你会发现其实没有鼠标，程序员也能够很嗨皮地工作。
